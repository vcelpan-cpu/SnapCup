# Kondex Order Bot

Telegram-бот для приёма ежедневных заказов от кофеен на кухню кондитерской.
Стек: aiogram 3, PostgreSQL (SQLAlchemy async), APScheduler, Railway — тот же паттерн, что у VeeChe.

## Логика

**Кофейни (смешанная регистрация)**
- `/start` → кофейня вводит название → заявка уходит в админ-чат с кнопками ✅/❌.
- Пока не подтверждена — статус `pending`, бот вежливо отказывает в командах.
- Админ подтверждает вручную (это и есть контроль допуска — по сути «фиксированный список», но без ручного ввода tg_id: точка сама стучится, а вы решаете кого пускать).
- `/block <id>` — закрыть доступ точке в любой момент.

**Каталог (единый для всех)**
- `/add_product Название; единица; цена` — добавить позицию.
- `/catalog` — список с id и статусом.
- `/toggle_product <id>` — включить/выключить позицию (не показывается кофейням, если выключена).

**Заказ**
- `/order` — инлайн-каталог с ➖/➕ по каждой позиции, кнопка «Оформить».
- Заказ живёт как «черновик» на сегодняшнюю дату, редактируется сколько угодно раз через `/order` до кат-оффа.
- `/status` — что уже набрано.

**Кат-офф и отчёт на кухню**
- В `CUTOFF_HOUR:CUTOFF_MINUTE` (по умолчанию 15:00 МСК) все заказы на сегодня блокируются автоматически, и в админ-чат уходит отчёт **в двух форматах сразу**:
  1. Сводный список — сколько чего испечь суммарно.
  2. Разбивка по каждой кофейне — для фасовки/доставки.
- `/report` — тот же отчёт вручную в любой момент, без блокировки заказов.

## Деплой на Railway

1. Создать проект, подключить PostgreSQL плагин (даст `DATABASE_URL`).
2. Задать переменные окружения (см. `.env.example`):
   - `BOT_TOKEN` — от @BotFather
   - `ADMIN_CHAT_ID` — id чата/группы кухни (узнать через @userinfobot или логи бота)
   - `ADMIN_USER_IDS` — telegram id людей, которые могут подтверждать кофейни и править каталог
   - `TZ_NAME`, `CUTOFF_HOUR`, `CUTOFF_MINUTE` — по умолчанию Europe/Moscow, 15:00
3. Деплой из репозитория — `Procfile` уже указывает `worker: python bot.py`.
4. Таблицы создаются автоматически при первом запуске (`init_db()`).
